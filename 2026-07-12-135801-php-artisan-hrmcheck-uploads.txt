в—Ҹ The --null-missing option is added and working.

  What it does

  hrm:relocate-uploads --null-missing sets the DB field to NULL for any
  reference whose file can't be located anywhere on disk. Key behaviors:

  - Only nulls the truly-unrecoverable (no basename match anywhere). Files that
  are just in a legacy folder still get moved, not nulled вҖ” so running it now
  correctly targets exactly theВ 770.
  - --dry-run compatible вҖ” --null-missing --dry-run reports "would be nulled"
  and changes nothing.
  - Ambiguous matches are never nulled вҖ” they're still reported for manual
  review.
  - Uses a keyed query update (no model events), so it only touches the one
  column.
